## 1. NEAR Protocol

### 戦略ポジション: "ユーザー所有のアジェンティックコマース"

#### 核心コンセプト

NEAR AIとNEAR Intentsを組み合わせた「ユーザーが意図を宣言し、エージェントが競合して実行する」モデル。

```
ユーザー「ポートフォリオをリバランスしたい」
    ↓
NEAR AI エージェント群（複数が競合入札）
    ↓
NEAR Intents（最善の実行経路で決済）
    ↓
結果: ユーザーは「何をするか」を宣言するだけ
```

#### NEAR Intents の仕組み

- インテント（アウトカム記述）を宣言
- ソルバーが最適実行経路を競争入札
- 決済はNEARで即時確定
- **200,000倍成長（2025年中）** — 旧NEARの国際送金ユースケースから爆発的拡大

#### マルチレベル委任

- エージェントがエージェントを雇う多段階委任に対応
- 「法的契約と裁判所をAIエージェントの調整が代替する」がNEARのビジョン
- OpenClaw・Claude・Codex等主要フレームワークと互換

#### 2026年ロードマップ

| 項目 | 内容 |
|---|---|
| TPS目標 | 実用的1,000,000 TPSの達成 |
| ガバナンス革新 | AI代表ノードが参加する新ガバナンスモデル（人間投票 + AIエージェント代表の混成） |
| x402統合 | NEAR Intent × HTTP 402 のネイティブ統合 |

#### TONへの示唆

| NEAR動向 | TONへの示唆 |
|---|---|
| NEAR Intents（ソルバーネットワーク） | TONにソルバー市場が存在しない → Intent Layer実装の機会 |
| エージェントのマルチレベル委任 | TAMPが萌芽だが、NEAR水準の実用化には遠い |
| AIガバナンス（エージェント投票参加） | TONガバナンスへのAI統合は未着手 |

## 2. Sui

### 戦略ポジション: "DeFi高度化 + 並列処理 + AIエージェントのポジション管理"

#### ネットワーク指標

| 指標 | 数値 |
|---|---|
| TVL推移 | $40M (2023) → **$600M** (Mar 2026) |
| 月次アクティブ開発者 | **954人**（Aptosの2倍） |
| 成長率（TVL） | 3年で15倍 |

#### アーキテクチャの差別化要因

**オブジェクトセントリック並列処理**

```
ETH/EVM: グローバルステート → シリアル実行（コンフリクト多）
Sui: オブジェクト所有者ベース → 独立オブジェクトを並列実行可

AIエージェントが複数ポジションを並列管理する場合:
- EVM: 前の操作完了を待つ → レイテンシ増加
- Sui: 独立ポジションを並列処理 → スループット向上
```

#### 主要プロダクト・動向（2026）

| プロダクト/出来事 | 内容 |
|---|---|
| **USDsui** | SUI + Stripeのステーブルコイン。発行収益でSUIトークン買い戻し |
| **DeepBook Margin** | SUI上の高度なDeFiマージン取引 |
| **eSui Dollar** | SUI固有のネイティブステーブルコイン |
| **Hashi** (2026-03) | BTC担保ステーブルコインレンディングプリミティブ |
| **DeFi Moonshots Program** | ビルダー支援・インセンティブプログラム |

#### TONへの示唆

| Sui動向 | TONへの示唆 |
|---|---|
| オブジェクト並列処理 | TONのactor modelも並列性は高いが設計哲学が異なる |
| BTC担保レンディング（Hashi） | TON上のGifts/Stars担保レンディングの参考アーキテクチャ |
| USDsui（Stripe連携） | TON Pay 2.0のStripe連携と競合する方向性 |
| 月次アクティブ開発者954人 | TONはハッカソン200件規模 → コミュニティ成熟度でSuiに劣後 |

## 3. Aptos

### 戦略ポジション: "AI・DePIN・次世代アプリのパフォーマンスL1"

#### 主要動向（2026）

| 出来事 | 内容 | 重要度 |
|---|---|---|
| **Microsoftとの戦略的パートナーシップ** | AI×ブロックチェーンツール共同開発 | ★★★ |
| **AIP-113** (Feb 2026) | SuiウォレットがAptosと相互接続可能に → Move系エコシステム統合 | ★★ |
| Moveの型システム | 金融アプリ・コンプライアンス向けセキュリティ（リソース所有権が言語レベルで保証） | ★ 技術 |

#### Microsoft連携の意義

MicrosoftがAIツールをAptos上で展開するとすれば:
- EnterpriseレベルのAIエージェントがオンチェーン資産を操作
- Microsoft 365 + Aptosウォレットの統合可能性
- FortunE 500企業のWeb3エントリーポイントとなる可能性

#### AIP-113: Move系統合

SuiとAptosが相互接続されることで:
- Move言語エコシステムが統合され開発者プールが拡大
- クロスチェーン流動性が自然に形成
- EVM覇権への対抗軸が強化

## 4. Trust Wallet TWAK（マルチチェーン横断）

Aptos / Sui / NEAR / TON 全てに対応するAIエージェントウォレットキット。

| 機能 | 詳細 |
|---|---|
| 対応チェーン | 25+（TON・SUI・Aptos・NEAR・ETH・Solana含む） |
| クロスチェーンスワップ | 単一UIで複数チェーン間移動 |
| DCA自動化 | 定期積立をエージェントが実行 |
| x402ゲーティング | マイクロペイメント対応コンテンツアクセス |

**TONにとっての意味:** TWAKがTONを対応チェーンに含むことで、TONユーザーが他チェーンアプリにアクセスできる。逆にTWAK経由でTONへの流入も期待できる。
